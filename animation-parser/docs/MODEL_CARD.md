# Model card: `ivone-intent-linear-int8-1.0.0`

## Purpose

This model classifies animation-command intent as supporting evidence for a deterministic parser. It is deliberately not responsible for JSON generation, entity resolution, numeric product semantics, or planner execution.

## Architecture

- 256-dimensional signed hashed feature vector.
- Word unigrams, word bigrams, and character trigrams.
- 17-way linear softmax classifier.
- INT8 weight-only QDQ ONNX graph; float input/output.
- Deterministic argmax; no sampling.
- Model size: see `reports/model-evaluation.json` (approximately 5 KB).

The hybrid was chosen over a generative model because the domain ontology is closed, scene references must never be invented, exact same-input determinism matters, and the deterministic layer is required even with a neural decoder. Joint BERT intent/slot research establishes encoder-based intent and slot prediction as a suitable task formulation, while this dataset's held-out evaluation did not justify shipping a transformer.

## Training and evaluation

Training uses deterministic full-batch-order SGD implemented in `scripts/train.mjs`; it has no Python dependency. Training data is generated by `scripts/generate-dataset.mjs`. Exact executed metrics are in:

- `reports/model-evaluation.json`
- `reports/evaluation.json`

The standalone linear classifier reaches only the held-out accuracy recorded in `reports/model-evaluation.json` (about 39% on deliberately unseen template families). It is therefore advisory and is not reliable enough to create actions by itself. The shipped hybrid's deterministic parser metrics are the production acceptance metrics; this measured gap is why a neural decoder was not trusted with JSON or references.

## Runtime

The browser worker uses ONNX Runtime Web. It attempts WebGPU, catches unsupported graph/provider failures, and recreates the session with WASM. Official ONNX Runtime documentation describes the WebGPU/WASM providers and recommends WASM for very lightweight models:

- https://onnxruntime.ai/docs/tutorials/web/
- https://onnxruntime.ai/docs/tutorials/web/ep-webgpu.html
- https://onnxruntime.ai/docs/performance/model-optimizations/quantization.html

## Intended use

English animation authoring prompts within the schema ontology. The deterministic parser remains authoritative.

## Out-of-scope use

General conversation, safety classification, arbitrary 3D physics, code generation, multilingual parsing, or autonomous scene invention.

## Limitations

- Hashed features can collide.
- The grammar parser has stronger guarantees than the classifier and may disagree on novel colloquialisms.
- English only.
- Complex nested temporal logic and multi-sentence ellipsis can require a warning or manual rephrasing.
- This compact model does not replace a learned token-level slot model if future evaluation shows a clear need.
